INVITE SYSTEM ARCHITECTURE
=================================

POST /api/sites/:id/invites
  body: { email: string }
  → validates email, checks for duplicate (409 if exists)
  → sends HTML email via Resend (RESEND_API_KEY secret already in CF)
  → inserts into site_invite table
  → returns 201 { invite: { id, email, invitedBy, createdAt } }

GET /api/sites/:id/invites
  → returns { invites: [...] } newest first

Both routes require session + site ownership.
Email sent from: hello@dreamysuite.com
Editor link sent: https://dreamysuite.com/sites/{siteId}
