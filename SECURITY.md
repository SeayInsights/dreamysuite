# Security Policy — DreamySuite

## Reporting a Vulnerability

Email **dannis.seay@twinrootsllc.com** with subject `[DreamySuite] Security Report`.

We aim to respond within **30 days**.

## What to Report

- Credential or token exposure in config or environment files
- Authentication/authorization bypass vulnerabilities
- Injection vulnerabilities (SQL, header, path traversal)
- Dependency CVEs in package.json

## What NOT to Report

General bugs belong in GitHub Issues.
